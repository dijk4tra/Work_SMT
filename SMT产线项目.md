### SMT 检测设备日志索引与故障检索系统

_2025/12 - 2026/05_

**项目简介：**
面向 SMT 检测工位、测试台和产线工控机的运维排障场景，对已归档的运行日志、报警日志和测试记录进行增量解析和索引构建，提供错误码查询、结构化条件筛选、正文关键词检索和原始日志回读功能，解决人工逐文件排查效率低和历史故障定位困难等问题。

**技术环境：**
C++11、Linux、Workflow / Wfrest、SRPC、Protobuf、MySQL、Redis、倒排索引、BM25、SLRU Cache、CMake、Git

**主要工作：**

1.  将 HTTP 接入层与日志检索服务拆分为独立进程，基于 Wfrest 对外提供错误码、关键词、异常日志和日志详情查询接口，使协议解析、参数校验与索引查询逻辑能够独立维护。

2.  使用 Protobuf 定义检索条件、分页参数、日志摘要和查询结果，通过 SRPC 完成接入服务与检索服务之间的调用；设置 RPC 超时时间，并将网络错误、调用超时和服务端异常统一转换为 HTTP 错误响应。

3.  将错误码、模块名、日志等级、设备编号等结构化字段写入文档元数据，并为日志正文构建倒排索引；多关键词 AND 查询时优先处理低文档频率关键词，使用 Top-K 小顶堆筛选高分结果，减少候选文档数量和全量排序开销。

4.  对新增日志按批次构建增量索引，构建完成后再将该批索引加入查询范围，避免未完成的索引数据参与检索。

5.  使用 BM25 计算正文关键词相关性，并结合错误码、模块名精确命中和日志等级进行排序，使错误码及模块匹配的日志优先展示。

6.  在索引中记录日志所属文件及对应的字节 offset 和 length，检索命中后使用 pread 按偏移回读原始日志内容，避免将大量日志正文重复保存到 MySQL。

7.  采用本地 SLRU Cache 与 Redis 分层缓存：本地缓存热点日志详情、错误码和文件元数据，通过 Probation / Protected 分区降低一次性历史查询造成的缓存污染；Redis 缓存关键词和错误码查询的日志 ID 列表及空结果，近期时间段使用短 TTL、已结束的历史时间段使用较长 TTL，减少对倒排索引、MySQL 和归档文件的重复访问。

### SMT 检测工位数据采集与归档管理平台

_2025/01 - 2025/11_

**项目简介：**
面向 SMT 检测工位、测试台和产线工控机的数据采集场景，接收采集程序上传的检测结果、测试报告、NG 图片、设备导出文件和运行记录，完成设备认证、分片上传、文件归档、元数据管理、设备状态维护和历史数据查询，解决检测数据分散存储和历史追溯困难等问题。

**技术环境：** C++11、Linux、Workflow / Wfrest、MySQL、Redis、OpenSSL、POSIX 文件 I/O、mmap、CMake、Git

**主要工作：**

1.  基于 Wfrest 实现设备认证、心跳上报、上传会话创建、文件分片上传和历史记录查询接口；使用 Workflow 组织设备信息校验、Redis 上传状态查询和 MySQL 元数据访问任务，并在任务回调中统一返回业务错误码。

2.  基于 OpenSSL 实现 HMAC-SHA256 请求签名校验，将设备编号、时间戳、请求标识和请求体摘要纳入签名内容；分片请求签入 uploadId 和分片编号，并结合时间窗口与 Redis 请求标识去重限制重放请求。

3.  对大文件采用分片上传，使用 posix\_fallocate 预分配临时文件空间，并通过 pwrite 按偏移写入分片；使用 Redis Bitmap 记录已完成的分片编号，采集端断线重连后查询并补传缺失分片，实现乱序上传和断点续传。

4.  所有分片上传完成后，采用窗口化 mmap 分段映射临时文件并增量计算 SHA-256，与采集端提交的文件大小和哈希值进行完整性校验。

5.  文件校验通过后，在同一文件系统内通过 rename 将临时文件原子移动至正式归档目录；以 uploadId 和分片编号作为分片请求的幂等键，结合 Redis Bitmap 判断分片完成状态；归档记录通过数据库唯一约束避免完成接口重试造成重复入库。

6.  设计文件元数据表，记录设备、工位、工单、产品 SN、文件类型、相对路径、文件大小、哈希值和上传时间；围绕设备编号与上传时间、工单号与产品 SN 等高频查询组合设计联合索引，减少历史追溯查询中的全表扫描。

7.  使用 Redis 保存设备心跳、上传会话和分片完成状态，并设置相应过期时间；定期清理超时未完成、校验失败和归档失败的临时文件及 Redis 状态，避免异常任务长期占用磁盘和缓存空间。
