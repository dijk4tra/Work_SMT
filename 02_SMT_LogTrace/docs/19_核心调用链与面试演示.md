# 核心调用链与面试演示

## 1. 增量索引调用链

`archive_file archive_id 游标 → 文件大小/SHA-256 回验 → parser_profile → PARSED 工件 → Term/Posting/
Document/File 表 → fsync → rename → MySQL READY → 内存快照交换`。

构建中间状态不进入查询；重启时只加载数据库登记且 manifest 校验通过的 READY Segment。

## 2. 查询调用链

`Nginx → Wfrest Gateway 鉴权/校验 → SRPC → Redis 查询页 → 本地倒排索引 → 低 DF AND → BM25 +
业务权重 → Top-K → Protobuf → HTTP`。Redis 失败时跳过缓存，不能改变结果。

## 3. 详情调用链

`doc_id → 当前快照 Document/File 表 → SLRU → archive relative_path 安全解析 → pread(offset,length)`。
正文不写入 MySQL 或 Redis，索引只保存定位信息。

## 4. 推荐演示

依次演示未认证 401、关键词查询、异常列表、错误码知识、详情原文、Redis Key 快照版本、停止 Redis
后的正确查询，以及损坏 Segment 导致的明确启动失败。所有性能数字须说明是固定本机仿真。
