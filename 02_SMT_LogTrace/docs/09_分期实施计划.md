# 第一生产版本分期实施计划

## 1. 总体规则

项目采用六期连续生成。每一期都提供实际可运行能力，不创建未实现的空服务、固定成功响应或
需要用户补写的代码。六期全部达到门槛后才标记 `v1.0.0`。

## 2. 第一期：工程、双进程与健康链路

状态：复验通过，待用户确认。

生成：

- CMake、C++11、Protobuf/SRPC 生成、质量选项；
- 严格配置、日志、请求标识和统一 HTTP 响应；
- Workflow MySQL/Redis 客户端和目录检查；
- `logsearch_server`、`logtrace_gateway`；
- `Health` RPC、`/health/live`、`/health/ready`；
- `smt_logtrace` 初始迁移、开发种子和测试基础。

门槛：Debug/Release 构建；配置与目录负向测试；RPC/HTTP 就绪链路；真实 MySQL/Redis 集成测试；
无搜索业务占位接口。

当前证据：Debug/Release 构建通过；数据库迁移首次执行与重复执行通过；开发种子重复执行通过且
无弃用告警；两个 MySQL、Redis、无效端口、SRPC/HTTP 就绪链路和 Search Server 停止后的错误
映射均已验证；CTest 17/17 通过。第一期已达到发布门槛，等待用户确认后再进入第二期。

## 3. 第二期：增量消费与日志解析

- 按 archive_id 扫描上游 RUNTIME_LOG/TEST_REPORT；
- parser_profile、`kv_runtime_v1`、`fct_csv_v1`；
- 文档元数据、精确 offset/length 和 SHA-256 回验；
- 批次与归档状态、失败记录和明确重建命令；
- 固定三线日志生成器和解析测试。

门槛：正常、未知配置、损坏、长行、CRLF、UTF-8 和 CSV 边界均有证据；本期不对外搜索。

## 4. 第三期：倒排索引与原子发布

- Term 字典、posting、文档和文件表；
- 不可变 Segment、manifest、摘要、fsync 和 rename；
- READY 发布、启动恢复、孤儿/临时 Segment 处理；
- 查询快照加载及 `pread` 原文详情能力。

门槛：构建中断不影响旧查询，重启可恢复 READY Segment，原文回读字节完全一致。

## 5. 第四期：检索算法与 HTTP/RPC 业务链

- 低 DF 优先 AND 求交、结构化过滤、BM25 和业务权重；
- Top-K 小顶堆、分页上限和稳定排序；
- SearchLogs、ListAnomalies、GetLogDetail、GetErrorCode RPC；
- 四类 HTTP 接口、Operator Token 和错误映射。

门槛：固定相关性集合排序正确，RPC 超时/网络/业务错误映射可重复验证。

## 6. 第五期：SLRU、Redis 与性能

- 线程安全 SLRU，Probation/Protected 晋升和降级；
- 日志详情、错误码和文件元数据本地缓存；
- Redis 查询 ID 列表和空结果 Cache-Aside；
- 快照版本 Key、近期/历史 TTL、Redis 故障回退；
- 并发查询和固定百万文档负载。

门槛：缓存命中、晋升、淘汰、空结果和 Redis 不可用均保持结果正确，无不可解释 5xx。

## 7. 第六期：跨项目闭环与终板

- DataStream -> Collector -> 归档 -> 增量索引 -> HTTP 查询完整 E2E；
- Nginx、systemd、logrotate、备份恢复和升级回滚；
- 格式、静态检查、覆盖率、Sanitizer、Valgrind 和压力报告；
- 故障演练、调用链、简历证据矩阵、学习路线和面试材料。

门槛：简历七条主要工作全部有源码、测试和可重复命令，不把仿真指标外推为真实工厂容量。
