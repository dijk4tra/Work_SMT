# 第一生产版本分期实施计划

## 1. 总体规则

项目采用六期连续生成。每一期都提供实际可运行能力，不创建未实现的空服务、固定成功响应或
需要用户补写的代码。六期全部达到门槛后才标记 `v1.0.0`。

## 2. 第一期：工程、双进程与健康链路

状态：复验通过，用户已确认。

生成：

- CMake、C++11、Protobuf/SRPC 生成、质量选项；
- 严格配置、日志、请求标识和统一 HTTP 响应；
- Workflow MySQL/Redis 客户端和目录检查；
- `logsearch_server`、`logtrace_gateway`；
- `Health` RPC、`/health/live`、`/health/ready`；
- `smt_logtrace` 初始迁移、开发种子和测试基础。

门槛：Debug/Release 构建；配置与目录负向测试；RPC/HTTP 就绪链路；真实 MySQL/Redis 集成测试；
无搜索业务占位接口。

当前证据：Debug/Release 构建通过；数据库迁移首次执行与重复执行通过；开发种子重复执行通过且
无弃用告警；两个 MySQL、Redis、无效端口、SRPC/HTTP 就绪链路和 Search Server 停止后的错误
映射均已验证；第一期 CTest 17/17 通过。用户已确认第一期，并授权进入第二期。

## 3. 第二期：增量消费与日志解析

- parser_profile、`kv_runtime_v1`、`fct_csv_v1`；
- 文档元数据、精确 offset/length 和 SHA-256 回验；
- 批次与归档状态、失败记录和明确重建命令；
- 固定三线日志生成器和解析测试。

门槛：正常、未知配置、损坏、长行、CRLF、UTF-8 和 CSV 边界均有证据；本期不对外搜索。

当前证据：已实现后台轮询、单次扫描、批次重建、两种解析器、大小与 SHA-256 回验、
精确字节范围、文件级原子失败和 `PARSED` 工件发布；`002_parsed_batch.sql` 已迁移；
Debug/Release CTest 29/29 通过。倒排索引、Segment 和搜索接口未进入本期。
用户已确认第二期，并授权进入第三期。

## 4. 第三期：倒排索引与原子发布

状态：实现与复验通过，用户已确认。

- Term 字典、posting、文档和文件表；
- 不可变 Segment、manifest、摘要、fsync 和 rename；
- READY 发布、启动恢复、孤儿/临时 Segment 处理；
- 查询快照加载及 `pread` 原文详情能力。

门槛：构建中断不影响旧查询，重启可恢复 READY Segment，原文回读字节完全一致。

当前证据：已实现固定小端二进制格式、Term 字典、Posting、文档/文件表、manifest 摘要、
`fsync + rename`、单语句 MySQL 原子 READY/INDEXED 发布、BUILDING 恢复、孤儿忽略、快照交换和
循环 `pread` 回读；`003_ready_segment.sql` 已迁移；Debug/Release CTest 34/34 通过。
搜索 RPC/HTTP、BM25 和 Top-K 未进入本期。
用户已确认第三期，并授权进入第四期。

## 5. 第四期：检索算法与 HTTP/RPC 业务链

状态：实现与复验通过，用户已确认。

门槛：固定相关性集合排序正确，RPC 超时/网络/业务错误映射可重复验证。

当前证据：已实现低 DF 优先 AND、跨 Segment BM25、结构化过滤、业务权重、Top-K 小顶堆、
稳定分页、四个 RPC 和四个 HTTP 接口，并通过 Operator Bearer Token 保护业务路由。
Debug/Release CTest 39/39 通过，固定样本已验证认证、搜索、异常列表、原文详情和错误码知识链路。
用户已确认第四期，并授权进入第五期。

## 6. 第五期：SLRU、Redis 与性能

状态：实现与复验通过，待用户确认。

- 线程安全 SLRU，Probation/Protected 晋升和降级；
- 日志详情、错误码和文件元数据本地缓存；
- Redis 查询 ID 列表和空结果 Cache-Aside；
- 快照版本 Key、近期/历史 TTL、Redis 故障回退；
- 并发查询和固定百万文档负载。

门槛：缓存命中、晋升、淘汰、空结果和 Redis 不可用均保持结果正确，无不可解释 5xx。

当前证据：已实现线程安全 SLRU、日志详情/文件元数据/错误码知识本地缓存、Redis 查询 ID 页、
规范化快照版本 Key 和四类 TTL。单元测试覆盖晋升、降级、淘汰、并发容量、Key 和损坏值；真实
Redis E2E 覆盖写入、命中和损坏回退，运行期 Redis 断开时 HTTP 仍返回本地索引结果。固定内存负载
完成 100 万文档、8 并发查询并校验稳定结果。Debug/Release 和完整 CTest 结果见第五期报告。
