# 第五期开发报告

完成日期：2026-07-14。

## 1. 阶段结论

第五期“SLRU、Redis 与性能”已完成实现与复验，当前状态为“复验通过，待用户确认”。本期只在
Search Server 内增加分层缓存，不改变 HTTP/RPC 契约、索引发布规则或一期只读边界。

## 2. 本地 SLRU

- 新条目先进入 Probation，第二次命中晋升 Protected；
- Protected LRU 溢出时降级回 Probation，Probation LRU 溢出时淘汰；
- `std::mutex` 保护链表、哈希表和晋升过程，更新已有 Key 不增加容量；
- 日志详情和文件元数据 Key 包含快照版本，避免索引重建后复用旧定位；
- 超过 `max_detail_bytes` 的原文不缓存，避免单条异常日志污染内存；
- 错误码知识缓存独立于索引快照，匹配日志列表仍按快照查询。

## 3. Redis Cache-Aside

Redis 值只保存当前页的 `doc_id`、对应分数和 `total_hits`，命中后从同一 READY 快照恢复结构化摘要，
因此不会复制日志正文，也不会改变 BM25 排序。Key 包含最大 READY `batch_id` 和规范化查询 SHA-256，
旧快照 Key 依靠 TTL 自然失效。

默认 TTL：近期非空 30 秒、近期空结果 10 秒、历史非空 600 秒、历史空结果 300 秒。Redis GET、
SETEX、网络或载荷解析失败时均执行本地索引；没有新增重试或错误结果兜底。
