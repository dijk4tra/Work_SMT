# 日志检索三线仿真业务基线

## 1. 拓扑

沿用一期三条产线、三台 IPC 和九台 SPI/AOI/FCT 设备。二期不另建一套设备主数据，所有生产
位置和归档路径来自一期 `archive_file`。

## 2. 数据来源

| 来源 | 文档粒度 | 主要字段 |
|---|---|---|
| 设备运行日志 | 一条物理日志行 | 时间、等级、模块、设备、SN、错误码、正文 |
| IPC 采集日志 | 一条物理日志行 | 时间、等级、collector、源设备、事件、请求标识 |
| FCT 测试报告 | 一个测试点 CSV 行 | 测试项、上下限、实测值、单位、结果 |

WARN/ERROR、非空错误码或测试点 NG 统一属于异常记录，但保留其真实来源类型，不能把测试记录
伪装成设备报警。

## 3. 本地验收规模

第一生产终板的固定本机规模暂定为：

- 3 条线、9 台设备、3 台 IPC；
- 7 个模拟生产日；
- 至少 1,000,000 条可检索文档；
- 错误/告警比例约 3% 至 8%；
- 关键词查询 1 至 4 个 term，AND 语义；
- 单次返回默认 50、最多 100，结果窗口最多 1000；
- 增量批次最多 100 个归档文件或 100,000 个文档，以先达到者为准。

这些值用于本机学习和性能回归，不代表真实工厂容量。第五、六期会根据实际压测调整，但必须同步
更新本文，不能只修改测试参数。

## 4. 时间与缓存假设

- 最近 2 小时视为活跃时间段；
- 活跃查询缓存 30 秒，空结果 10 秒；
- 已结束历史范围缓存 10 分钟，空结果 5 分钟；
- 查询缓存 Key 必须包含索引快照版本；
- 本地 SLRU 容量以可配置条目数限制，不缓存无界原始文件。

## 5. 验收场景

1. 一期新增归档在一个轮询周期后进入新 Segment；
2. 构建中断时旧快照继续查询，未完成 Segment 不可见；
3. 错误码和多关键词查询返回预期文档；
4. 低 DF 求交与 Top-K 排序结果可重复；
5. `pread` 回读内容与原归档指定字节完全一致；
6. Redis 不可用时结果正确，仅延迟增加；
7. Search Server 重启后从 READY Segment 恢复；
8. Gateway 能区分 RPC 超时、网络失败和服务端不可用。
