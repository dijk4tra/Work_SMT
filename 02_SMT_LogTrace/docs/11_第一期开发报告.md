第一期设计文档和代码已经完成，工程能够在 C++11 下生成 Protobuf/SRPC 代码并构建
`logsearch_server` 与 `logtrace_gateway`。本期交付的是真实健康链路和基础设施启动检查，没有

## 3. 当前架构

```text
smt_datastream.archive_file + archive_root（只读）
                       |
                       v
               logsearch_server
        依赖检查 / 后续增量索引 / 查询引擎
                       ^
                  SRPC + Protobuf
                       |
               logtrace_gateway
           Wfrest HTTP / 错误映射
```

第一期只实现图中的进程、通信和依赖边界。增量发现、日志解析、Segment、检索算法和缓存按照第二期
至第五期逐步加入，最终在第六期完成与 DataStream 的跨项目闭环。

## 4. 关键实现约束

- 二期对 `smt_datastream` 和一期归档文件系统只读，不修改一期元数据与文件；
- 二期状态写入独立数据库 `smt_logtrace`，Search Server 启动时不自动迁移；
- MySQL 密码只从 `SMT_LOGTRACE_SOURCE_MYSQL_PASSWORD` 和
  `SMT_LOGTRACE_STATE_MYSQL_PASSWORD` 读取，不写入 JSON、源码或日志；
- Gateway 到 Search Server 的健康 RPC 使用明确超时和零重试；
- Search Server 启动时严格检查两个 MySQL、Redis、归档目录和索引目录，任一必要依赖失败即退出；
- Gateway 启动前必须确认 Search Server 已就绪，不提供离线模式或静默降级；
- 第一阶段 Protobuf 只声明已实现的 `Health` RPC，避免业务占位接口。
