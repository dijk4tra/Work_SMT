## 3. 跨项目闭环

`cross_project_e2e_test.py` 使用独立临时源库、状态库、归档和索引目录，真实启动一期 DataStream 与
Collector。Collector 上传 AOI `RUNTIME_LOG` 后，二期 `scan-once` 解析 1 个文档，`build-once`
发布快照版本 1，最终 HTTP 关键词查询命中 `INSPECTION_NG`。脚本结束后删除临时数据库。

## 4. 性能边界

百万文档数据为内存合成记录，不包含真实工厂磁盘冷启动、长文本分词、网络抖动或多租户竞争。
232 ms 只记录本次开发机运行，不能作为产线 SLA 或容量承诺。

## 5. 已知边界

- 单机本地文件系统与单 Search Server，不提供多实例共享索引或跨机房复制；
- Operator 仍是部署级单 Token，没有用户体系和 RBAC；
- Redis 查询缓存可丢弃，状态库和索引需要一致备份，一期原始归档由 DataStream 负责；
- systemd/Nginx 已完成配置语法验证，本轮未写入 `/opt`、`/etc` 或启动系统级服务。

至此简历中 LogTrace 七条主要工作均具备源码、测试和可重复命令，项目进入 `v1.0.0` 终板确认点。
