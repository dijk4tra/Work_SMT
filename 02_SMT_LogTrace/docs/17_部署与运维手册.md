# 部署与运维手册

## 1. 账户与目录

建议使用独立账户 `smt-logtrace`。二进制安装到 `/opt/smt-logtrace/bin`，配置和环境文件放在
`/etc/smt-logtrace`，索引目录为 `/var/lib/smt-logtrace/index`，日志目录为
`/var/log/smt-logtrace`。一期归档 `/var/lib/smt-datastream/archive` 只授予读取和目录遍历权限。

环境文件权限应为 `0640 root:smt-logtrace`，TLS 私钥为 `0600 root:root`。生产密码和 Operator
Token 必须由密钥系统注入，不得写入 JSON、systemd 单元或仓库。

## 2. 安装与启动

```bash
scripts/build.sh release
sudo install -m 0755 build-release/logsearch_server /opt/smt-logtrace/bin/
sudo install -m 0755 build-release/logtrace_gateway /opt/smt-logtrace/bin/
scripts/db.sh migrate --config /etc/smt-logtrace/logtrace.json
sudo systemd-analyze verify deploy/systemd/*.service
sudo systemctl daemon-reload
sudo systemctl enable --now smt-logsearch smt-logtrace-gateway
scripts/health_check.sh http://127.0.0.1:8081
sudo nginx -t && sudo systemctl reload nginx
```

启动顺序为 MySQL/Redis、DataStream、Search Server、Gateway、Nginx。Search Server 启动时严格加载
全部 READY Segment；缺失或损坏时拒绝启动，不能以空索引伪装就绪。

## 3. 日常观测

- `/health/live` 观察 Gateway 进程，`/health/ready` 观察 Search Server 及依赖；
- `journalctl -u smt-logsearch -u smt-logtrace-gateway --since today` 查看服务事件；
- 观察一期归档磁盘、二期索引磁盘、Redis 查询 Key 数量和 MySQL 批次失败状态；
- 告警关注 READY Segment 校验失败、索引批次持续 FAILED、RPC 超时和持续 5xx。

## 4. 备份与恢复

维护窗口先停 Gateway，再停 Search Server，避免备份时继续发布 Segment：

```bash
scripts/backup.sh --config /etc/smt-logtrace/logtrace.json /backup/logtrace-20260714
scripts/verify_backup.sh /backup/logtrace-20260714
```

恢复顺序为校验 SHA-256、恢复索引目录、导入状态库、启动 Search Server 并检查所有 READY Segment，
最后启动 Gateway。Redis 查询缓存不恢复；一期原始归档由 DataStream 的备份流程负责。

## 5. 升级与回滚

升级前记录二进制、配置和迁移版本并完成备份。先替换 Search Server，确认健康和固定查询，再替换
Gateway。未执行不可逆迁移时可恢复旧二进制链接；执行迁移后必须按版本化数据方案回滚，禁止手工
修改 `schema_migration`。索引格式不兼容时使用明确 rebuild 命令，不混用新旧 Segment。
