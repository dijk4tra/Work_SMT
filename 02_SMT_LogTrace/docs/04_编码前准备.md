# 编码前准备与终板契约

## 1. 技术基线

| 项目 | 约定 |
|---|---|
| 语言 | C++11 |
| HTTP | Wfrest |
| 异步任务 | Workflow |
| RPC | SRPC 0.10.2 |
| 序列化 | Protobuf 3.20.1 |
| 数据库 | MySQL：上游只读与二期状态库 |
| 缓存 | Redis + 本地 SLRU |
| 日志 | spdlog |
| JSON | nlohmann/json |
| 文件 I/O | 顺序读取、`pread`、`fsync`、`rename` |
| 测试 | GoogleTest、CTest、Python E2E |

## 2. 第一阶段范围

- CMake 和 Protobuf/SRPC 代码生成；
- `logsearch_server` 与 `logtrace_gateway`；
- `Health` RPC 和 `/health/live`、`/health/ready`；
- 严格 JSON/环境变量配置；
- 上游 MySQL、状态 MySQL、Redis 与目录检查；
- `smt_logtrace` 初始迁移和开发种子；
- 单元、集成和 HTTP/RPC 端到端测试基础。

第一阶段不创建搜索、详情或错误码业务路由，也不创建返回固定数据的索引接口。

## 3. 启动顺序

Search Server：加载配置、初始化日志、检查归档和索引目录、探测两个 MySQL 与 Redis、注册 RPC
服务、开始监听。任一必要条件失败时退出，不以空索引或内存假数据继续运行。第一阶段使用
Workflow 服务的阻塞停止语义；有界等待参数在出现真实在途索引任务的第二阶段再加入。

Gateway：加载配置、初始化日志、探测 Search Server `Health`、注册健康路由、开始监听。运行期间
Search Server 失联时 readiness 返回稳定错误，不返回成功。

## 4. 错误边界

- 配置、目录、数据库和 Redis 启动失败：进程启动失败；
- RPC 网络失败：Gateway 返回 `SEARCH_RPC_UNAVAILABLE`；
- RPC 超时：Gateway 返回 `SEARCH_RPC_TIMEOUT`；
- Search Server 依赖不可用：返回 `SERVICE_NOT_READY`；
- 未知程序错误不转换成成功或空结果。

## 5. 敏感值

两个 MySQL 密码和可选 Redis 密码只从环境变量读取。当前第一阶段没有业务查询认证接口，因此
不提前增加未使用的 Operator Token 配置；第四阶段加入业务 HTTP 路由时再增加并验证该契约。
