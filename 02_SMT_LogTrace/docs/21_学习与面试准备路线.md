# 学习与面试准备路线

1. 从 `archive_id` 游标、批次状态机和不可变 Segment 理解增量索引一致性。
2. 手写倒排表求交、BM25 公式和 Top-K 小顶堆，解释为什么先处理低 DF 关键词。
3. 结合 `offset/length + pread` 理解正文、元数据和索引的存储分工。
4. 画出 Wfrest → SRPC → Search Server 的异步调用链和超时错误映射。
5. 手写 LRU，再扩展为 Probation/Protected SLRU，解释历史扫描污染问题。
6. 分析 Redis Cache-Aside 的空结果、快照版本 Key、TTL 和故障回退。
7. 重复运行固定 E2E、百万文档负载、Sanitizer 和 Valgrind，并能解释指标边界。
8. 按故障演练文档排查 Redis、MySQL、Segment 和归档正文故障。
