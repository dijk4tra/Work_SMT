# 日志格式与解析契约

## 1. 输入边界

索引器只解析一期已归档的 `RUNTIME_LOG` 和 `TEST_REPORT`。文件正文是不可信外部输入，但路径、
大小和 SHA-256 来自一期已校验元数据。索引器在顺序读取时再次核对大小与摘要，不重新解释一期
上传认证过程。

## 2. 解析器选择

`parser_profile` 按 `device_id + file_type` 明确选择解析器。配置不存在或停用时，该归档进入
FAILED，错误码为 `PARSER_PROFILE_NOT_FOUND`。不通过扩展名或少量内容自动猜测厂商格式。

## 3. `kv_runtime_v1`

支持一期样本的单行 UTF-8 格式：

```text
2026-07-12T08:00:00.000+08:00 level=WARN module=inspection device=AOI-VT-01 \
station=ST-AOI-01 sn=CTRLMBA1-260712-000001 result=NG code=INSPECTION_NG temperature_c=42.9
```

规则：

- 首字段必须是带时区 ISO 8601 毫秒时间；
- 后续字段为以空格分隔的 `key=value`；
- `level`、`module`、`code` 等已知字段写入结构化元数据；
- 完整原始行仍参与正文分词；
- 未知 key 保留在正文，不进入首版结构化字段；
- 缺少非关键字段时使用明确 UNKNOWN/空值；时间或 UTF-8 非法时该行解析失败；
- 单行长度上限固定，超限不分段拼接成伪记录。

## 4. `fct_csv_v1`

支持一期 UTF-8 BOM CSV。文件头提供设备、工单、产品 SN、测试时间和总结果；表格区每个物理行
对应一个测试点，首版不允许字段内嵌换行。

测试点 NG 时规范化为：

- `level=ERROR`；
- `module_name=fct`；
- `error_code=FCT_LIMIT_FAIL`；
- 正文包含测试点、测试名、上下限、实测值、单位和结果。

该错误码是二期对测试记录的明确归一化字段，原始 CSV 行仍按 offset/length 回读，不修改文件。

## 5. 字节范围

Offset 从文件起始字节计算，Length 不包含行尾 `\n`，但包含行内 UTF-8 原始字节。CRLF 文件的
正文不包含结尾 `\r`。解析器必须在读取时记录精确字节位置，不能使用字符数代替字节数。

## 6. 失败处理

- 文件不存在、大小变化或 SHA-256 不符：不发布该文件文档，记录源完整性错误；
- 解析配置缺失：记录失败并允许后续批次继续；
- 单行格式错误：记录文件的失败行数；是否允许部分发布由第二阶段实现前固定；
- 索引写入失败：整个批次不 READY；
- 所有失败都保留稳定错误码和 archive_id，不打印完整原文。
