# 第五期开发报告

完成日期：2026-07-12。最终网络闭环状态：通过。

## 1. 交付内容

第五期新增 C++11 `collector_agent`，按受控配置管理三条 SMT 产线、九台 SPI/AOI/FCT 设备目录
及三台 IPC 绑定。采集程序只观察设备目录和同名 sidecar，不读取模拟器期望清单。

主要模块：

- `collector_config`：严格 JSON、目录映射、封口方式、spool 容量和密钥环境变量；
- `spool_store`：不可变 payload 快照、原子 JSON 状态、文件和目录 `fsync`；
- `http_uploader`：Workflow HTTP、设备 HMAC、创建/进度/分片/完成调用；
- `collector_app`：文件发现、去重、退避、断点续传和重启恢复状态机；
- `simulate_device_lifecycle.py`：三线并发文件行为和独立期望清单；
- `collector_e2e_test.py`：断网、退出重启、Redis 会话丢失和最终归档核对。

没有引入 SQLite、消息队列、目录插件体系、未知格式推断或无限重试。

## 2. 文件封口和 sidecar

九个设备映射分别配置三种明确封口信号：

1. `.tmp/.part` 写完后原子改为正式名；
2. 大小与纳秒 mtime 连续三个扫描周期不变；
3. 同名 `.done` 标志出现。

运行日志仅在配置为完成标志的目录中产生，不用短暂静默推断日志结束。正式文件的同名
`.meta.json` 只提供工单、产品 SN、文件类型、结果和产生时间；产线、工位、设备、IPC 和密钥
来自采集配置。零字节、损坏 sidecar 和封口后活动任务再次变化会持久化为明确 `FAILED`。

## 3. spool 一致性

任务身份由源规范路径、设备编号、封口大小、mtime 和 SHA-256 共同组成。相同内容位于不同业务
路径时不会合并。正式上传前先复制不可变 payload，并再次计算摘要，避免设备同名覆盖影响在途
任务。

状态文件通过临时文件写满、文件 `fsync`、原子 `rename` 和父目录 `fsync` 提交。启动时只删除
名称严格匹配的中断 `.json.tmp/.data.tmp`，损坏正式状态会阻止启动。`DONE` 状态提交后删除本地
payload 并 fsync 目录，保留小型状态文件用于重复发现去重。

## 4. 上传与失败分类

采集端每次请求生成新请求标识和设备 HMAC，不使用 Workflow 内建自动重试：

- 网络错误、超时、5xx、`UPLOAD_LIMIT_EXCEEDED` 和 `CHUNKS_INCOMPLETE` 进入封顶指数退避；
- `max_backoff_steps` 只限制指数增长，任务在长时间断网时仍按最大间隔保留和尝试；
- 其他 4xx 和损坏服务端响应进入永久失败；
- 已有 `upload_id` 时先查询服务端缺片，只补传 Bitmap 中缺失分片；
- `UPLOAD_NOT_FOUND` 表示 Redis 会话丢失，清除旧编号并为同一本地任务创建新会话；
- 完成成功后保存唯一 `archive_id` 再删除本地 payload。

## 5. 模拟场景

生命周期模拟器并发生成三条线的以下文件：

- SPI 检测 JSON，临时名原子改名；
- AOI NG 图片，多次追加并经过稳定窗口；
- FCT CSV 报告，追加后写完成标志；
- SPI 设备导出 ZIP，原子改名；
- 同一导出内容写入不同合法路径，验证不能只按摘要全局去重；
- FCT 运行日志，追加、关闭后写完成标志；
- 零字节和损坏 sidecar，期望为本地永久失败；遗留 `.tmp` 期望忽略；
- `--overwrite-existing` 可在封口后覆盖同名 AOI 文件；
- FCT-03 可配置大文件，用于延长分片窗口和故障注入观察时间。

期望 manifest 仅由 E2E 读取，用于核对应归档/应失败集合、大小和 SHA-256。

## 7. 最终闭环验收

`CollectorE2ETest` 已实际启动 DataStream、Collector、MySQL 和 Redis，验证：

1. DataStream 未启动时 collector 将 20 个应处理任务和 payload/失败状态持久排队；
2. collector 退出并重启后从 spool 恢复；
3. 创建服务端会话后停止 collector，并删除该 Redis 会话；
4. 再次启动 collector，观察旧会话 404、创建新会话并完成归档；
5. 18 个有效文件在 MySQL 和正式目录中的大小、SHA-256 与期望清单一致；
6. 两个异常文件保持 `FAILED`、遗留临时文件保持忽略，测试数据和 Redis 专用前缀最终清空。

首次验收脚本错误地要求 18 个任务同时处于 `UPLOADING`，与服务端单设备最多 2 个活跃会话的
合同冲突。修正为等待一个已持久化的活跃会话后强退并删除该 Redis 会话；最终仍严格要求全部
18 个有效文件完成大小、摘要和正文回验。修正后用例 4.22 秒通过，2 个异常文件保持 `FAILED`，
遗留 `.tmp` 保持忽略，Redis 专用前缀和测试工单数据均清理。

```bash
export SMT_DATASTREAM_MYSQL_PASSWORD='<mysql-password>'
export SMT_DATASTREAM_OPERATOR_TOKEN='e2e-operator-token-2026'
```
