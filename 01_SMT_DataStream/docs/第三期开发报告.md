# 第三期开发报告

代码完成日期：2026-07-11。

## 1. 完成范围

第三期已实现上传会话创建、磁盘水位和会话配额、临时文件预分配、乱序分片写入、分片摘要
幂等、Redis Bitmap 和进度查询。本期只产生预分配并写入完成的 `.part` 文件，不执行整文件
校验、正式 `rename`、MySQL 归档入库或历史查询。

## 2. 数据与配置变化

- 新增 `002_fct_collector_binding.sql`，扩展 FCT 工位并创建采集器—设备绑定表；
- 开发种子扩展为三条线、九个启用工位、九台启用设备和九条绑定；
- 旧 ICT 设备和工位保留但停用，不修改已执行 `001` 迁移；
- 上传配置新增磁盘水位、全局/设备/采集器会话数和全局预留字节上限；
- 模拟器默认生成三条线 SPI/AOI/FCT 文件，本期生成两周期共 63 个固定样本。

本机迁移和种子核对结果：3 条产线，SPI/AOI/FCT 各 3 个启用工位，9 条启用绑定。

## 3. 创建会话流程

1. 完成设备 HMAC 和防重放；
2. 严格解析文件元数据，校验设备工位和采集器绑定；
3. 检查临时文件系统字节及比例安全水位；
4. 使用 OpenSSL 生成 UUIDv4；
5. 在文件任务队列中创建 `.part` 并执行 `posix_fallocate`；
6. Redis Lua 清理过期配额记录，原子检查全局、设备、采集器和预留字节配额；
7. 写入会话 Hash、TTL 和配额 ZSet 后返回 HTTP 201。

Redis 创建失败或配额拒绝时删除刚创建的临时文件，不返回可用 `upload_id`。服务端不自动重试
文件、MySQL 或 Redis 操作。

## 4. 分片并发与崩溃边界

写盘前 Lua 原子登记 `(chunk_no, digest)`：

- 已完成且摘要相同直接返回幂等成功；
- 已登记不同摘要直接返回 `CHUNK_CONTENT_CONFLICT`，不会写盘；
- 相同摘要但 Bitmap 未完成时允许重新覆盖同一偏移，用于恢复写盘后未置位的中断窗口。

文件任务按 `upload_id` 使用 Workflow 命名队列串行执行 `pwrite` 写满循环。写盘成功后，第二个
Lua 脚本置 Bitmap，并刷新会话、摘要、Bitmap 和三类配额到期时间。若 Redis 完成标记失败，
客户端以相同内容重传并覆盖同一偏移。
