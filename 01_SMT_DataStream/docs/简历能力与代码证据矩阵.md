## 3. 已完成代码证据

### 3.1 设备认证与心跳

- `include/datastream/auth/crypto.h`、`src/auth/crypto.cpp`：SHA-256、HMAC-SHA256、规范串和
  常量时间比较；
- `src/auth/device_authenticator.cpp`：请求头、时间窗口、MySQL 设备状态和 Redis 防重放；
- `src/api/heartbeat_controller.cpp`：心跳模型、MySQL 更新时间和 Redis Hash+TTL；
- `tests/unit/crypto_test.cpp`：密码学已知向量；
- `tests/e2e/heartbeat_e2e_test.py`：正确签名、篡改、过期、重放、未知/停用设备、TTL 和日志脱敏。

验证命令：

```bash
export SMT_DATASTREAM_MYSQL_PASSWORD='123456'
export SMT_DATASTREAM_OPERATOR_TOKEN='e2e-operator-token-2026'
ctest --test-dir build --output-on-failure
```

第二期当前记录为 Debug 全量 31/31 通过。

### 3.2 元数据和索引

- `migrations/001_initial_schema.sql`：产线、工位、设备和归档元数据表；
- `archive_file` 唯一约束：`upload_id`、`relative_path`；
- 联合索引：设备时间、工位时间、工单与产品 SN 时间；
- `scripts/db.sh`：显式版本迁移和 checksum 防篡改。

目前只能说明“表和索引已经生成并迁移”，不能声称历史组合查询性能已经由接口和 `EXPLAIN`
验证；这部分在第四期补证据。
