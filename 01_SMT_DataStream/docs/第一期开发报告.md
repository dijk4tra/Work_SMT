# 第一期开发报告

完成日期：2026-07-11。

## 1. 完成范围

第一期已按 `分期实施计划.md` 完成工程骨架、配置、日志、基础设施客户端、存储目录、健康接口、
数据库迁移、开发种子和测试基础设施。没有提前生成第二期设备认证、心跳或上传业务占位路由。

## 2. 主要产物

- `CMakeLists.txt`：C++11、依赖检查、Debug/Release、测试、Sanitizer 和覆盖率选项；
- `include/datastream`、`src`：配置、统一响应、请求标识、日志、MySQL/Redis 客户端、目录检查、
  健康控制器、服务生命周期和进程入口；
- `migrations/000_schema_migration.sql`：迁移版本与 checksum 表；
- `migrations/001_initial_schema.sql`：产线、工位、设备和归档元数据表；
- `migrations/dev_seed.sql`：一条产线、三个工位和三台模拟设备；
- `scripts/db.sh`：显式 migrate/seed 命令，同版本 checksum 不一致时停止；
- `tests/unit`、`tests/integration`：配置、响应、请求标识、URI 编码、目录和真实服务连接测试；
- `.clang-format`、`.clang-tidy`：统一格式与首期静态检查规则。

## 4. 数据库核对

- 迁移版本：`000`、`001`；数据库 checksum 与迁移文件 SHA-256 一致；
- 表：`schema_migration`、`production_line`、`station`、`device`、`archive_file`；
- `archive_file` 的唯一键和联合索引均已创建；
- CHECK 约束覆盖启用状态、工位类型、文件大小、文件类型和检测结果；
- 开发设备：`SPI-ZM-01`、`AOI-VT-01`、`ICT-TRI-01`；
- 三台开发设备的 HMAC 密钥均为 32 字节，仅用于本地后续联调。

## 5. 当前边界

- 当前版本是第一期开发里程碑，不是生产终板；
- 只公开健康接口，设备认证、心跳、上传、归档完成、查询和清理按后续阶段生成；
- Redis 使用本机已恢复的无密码 `default` 用户；生产启用认证时通过可选环境变量注入；
- 服务启动时要求 MySQL、Redis 和文件目录全部可用，不提供离线模式和自动重试；
- 服务不会在启动时自动执行迁移，数据库变更只通过显式脚本完成。
