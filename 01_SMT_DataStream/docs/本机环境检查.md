## 构建注意项

- Workflow 与 Wfrest 没有安装 pkg-config 文件，CMake 需要在 `/usr/local/include` 和
  `/usr/local/lib` 查找，找不到时应明确配置失败，不在线下载另一份源码。
- GoogleTest 已系统安装，CMake 使用 `find_package(GTest REQUIRED)`，不通过 FetchContent
  联网下载第二份源码。
- Wfrest 未暴露可直接读取的版本号。构建记录可先保存实际头文件和库路径，不虚构版本。
- redis++ 头文件使用 C++17 标准库类型，项目不包含、不链接它；Redis 统一使用 Workflow。
- Redis 测试使用独立键前缀并只删除自身键，不执行 `FLUSHDB/FLUSHALL`。
- MySQL 集成测试使用独立测试表或事务清理，不修改未知的现有业务表。
- Nginx 已安装，但在第六期生成并评审站点配置前不替换系统默认配置。

## 环境准备结论

正式编码所需编译器、构建系统、服务、库、测试框架和检查工具均已就绪。第一期编码开始后
仍按分期计划生成并评审迁移 SQL，不能因为空数据库已经创建就跳过版本化迁移。生产环境需要
创建最小权限 MySQL 账号；本机开发阶段按用户提供的账号通过环境变量连接。
