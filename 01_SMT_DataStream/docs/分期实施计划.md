# 第一生产版本分期实施计划

## 1. 交付结论

项目采用六期连续生成。所有代码、SQL、配置、脚本、测试和部署文件均由 AI 生成；用户不需要
在阶段之间手工补实现。每一期都在上一期已验证基线上继续，禁止先放空类、空路由或固定成功
响应再声称阶段完成。

六期全部完成后的统一版本才是“SMT 检测工位数据采集与归档管理平台第一期生产终板”。
前五期是可验证的开发里程碑，不单独标记生产版。

## 2. 最终目录目标

目录会随对应阶段逐步生成，不在第一期一次性创建全部空文件。

```text
01_SMT_DataStream/
├── CMakeLists.txt
├── cmake/                  # 依赖查找和编译选项
├── conf/                   # JSON 配置、环境变量示例
├── collector/              # 第五期 C++ 参考采集程序
├── deploy/
│   ├── nginx/              # TLS 反向代理样例
│   ├── systemd/            # 服务单元样例
│   └── logrotate/          # 日志轮转样例
├── docs/                   # 设计、API、部署、运维和测试报告
├── include/datastream/
│   ├── api/
│   ├── archive/
│   ├── auth/
│   ├── common/
│   ├── config/
│   ├── device/
│   ├── storage/
│   └── upload/
├── migrations/            # MySQL 版本化迁移和开发种子数据
├── scripts/               # 构建、迁移、启动、健康与端到端验收
├── src/                    # 与 include 职责对应的 C++ 实现
├── tests/
│   ├── unit/
│   ├── integration/
│   └── e2e/
└── tools/                 # 模拟数据生成等开发工具
```

`common/` 只允许放至少两个业务模块共同使用的错误、时间、摘要和响应类型，不能演变成无边界
工具箱。MySQL、Redis 和文件系统各只有一个明确适配模块，不增加 Repository 接口加 Impl、
Factory 等没有第二实现的层级。

## 3. 第一期：工程骨架、基础设施与健康检查

状态：已完成（2026-07-11）。实现和验证证据见 `第一期开发报告.md`。

### 3.1 生成内容

- 根 CMake、依赖查找、Debug/Release、告警、ASan/UBSan、CTest 和覆盖率选项；
- `datastream_server` 可执行程序入口、信号处理和优雅停止框架；
- JSON 配置加载与一次性边界校验，环境变量读取但不保存敏感值；
- spdlog 初始化、统一业务错误类型、JSON 响应和请求标识传递；
- Workflow MySQL/Redis 任务创建与结果转换的具体适配代码；
- 临时/归档目录初始化、同文件系统检查和磁盘可写检查；
- `/health/live` 与 `/health/ready`；
- MySQL 迁移版本表，以及产线、工位、设备、归档文件的首版 DDL；
- LINE-01、SPI/AOI/ICT 工位和三台模拟设备的开发种子 SQL；
- GoogleTest 基础设施和第一批配置、错误映射、健康检查测试。

### 3.2 数据变化

生成迁移但只在明确执行迁移脚本时修改数据库：

- `schema_migration`：记录已成功执行的迁移版本；
- `production_line`；
- `station`；
- `device`；
- `archive_file`。

迁移脚本使用事务能力允许的 DDL 顺序，并在执行前检查版本；不在服务启动时偷偷自动改表。

### 3.3 完成门槛

- C++11 下所有终板依赖能共同编译链接；
- 缺密码、非法端口、目录不可写或临时/归档跨文件系统时启动明确失败；
- MySQL、Redis 和归档目录正常时 readiness 返回成功，任一依赖不可用时返回非就绪；
- 收到 SIGTERM 后停止监听并释放已初始化资源；
- 本期单元和集成测试全部通过，无业务接口占位实现。

## 4. 第二期：设备认证、防重放与心跳

状态：已完成（2026-07-11）。实现和验证证据见 `第二期开发报告.md`。

### 4.1 生成内容

- HMAC-SHA256 规范串构造、正文 SHA-256 和 OpenSSL 常量时间比较；
- 认证请求头解析、时间窗口校验、设备查询和启用状态判断；
- Redis `SET NX EX` 防重放；
- 设备心跳请求/响应模型、路由、Redis 实时状态和 MySQL 最近心跳更新；
- Operator Bearer Token 校验，用于后续只读运维接口；
- 认证日志字段，明确禁止输出签名、密钥、Bearer Token 和请求正文；
- 签名样例工具或测试辅助函数，供端到端脚本复用。

### 4.2 本期接口

- `POST /api/v1/devices/heartbeat`；
- 健康检查保持可用；
- Operator 认证中间处理能力生成，但不增加未定义的管理业务接口。

### 4.3 测试

- 正确签名、正文被篡改、错误设备、停用设备、过期时间和重复请求；
- 空正文摘要、大小写不规范摘要、缺失请求头和超长标识；
- Redis 不可用与 MySQL 不可用的错误区分；
- 心跳 TTL、在线状态计算和同一请求重放；
- 日志断言或审查确保敏感信息未输出。

### 4.4 完成门槛

模拟设备可以使用已登记密钥完成心跳；所有负向场景返回稳定 HTTP 状态和业务错误码，且没有
为了基础设施失败而返回“签名错误”的兜底。

## 5. 第三期：上传会话、分片与断点续传

状态：已完成（2026-07-12）。实现和最终并发验收证据见 `第三期开发报告.md`。

### 5.1 生成内容

- 新增迁移和开发种子，将目标仿真拓扑扩展为三条线，每线一个 IPC 及 SPI/AOI/FCT；
- 生成覆盖三线拓扑和六类业务数据的最小固定样本；第五期再扩展文件生命周期与连续场景；
- 采集器与设备绑定关系，以及上传时的 `collector_id` 授权校验；
- 文件类型、结果、上传状态和创建会话模型；
- 设备/工位归属、文件大小、摘要、分片大小和类型相关字段校验；
- `upload_id` 生成、临时安全路径和 `posix_fallocate`；
- 创建会话前的磁盘安全水位、全局与单设备在途配额；
- Redis 会话 Hash、Bitmap、分片摘要 Hash 及 TTL 管理；
- 会话级互斥，串行化同一上传的状态改变和文件写入；
- `pwrite` 写满循环、分片摘要校验、最后一片长度计算；
- 已完成同摘要分片的幂等成功与不同摘要冲突；
- 进度查询和缺失分片计算。

### 5.2 本期接口

- `POST /api/v1/uploads`；
- `PUT /api/v1/uploads/{upload_id}/chunks/{chunk_no}`；
- `GET /api/v1/uploads/{upload_id}`。

三个接口均使用设备 HMAC。设备只能操作自己创建的上传会话，不能只凭 `upload_id` 访问其他
设备会话。

### 5.3 测试

- 0 字节文件的产品决定：第一期明确拒绝，不创建零分片会话；
- 最小、最大、越界分片大小和 2 GiB 文件上限；
- 单分片、多分片、乱序、重复、不同摘要冲突和断线补传；
- `pwrite` 后未置位的重传恢复；
- 会话过期、设备不匹配、状态冲突和磁盘写失败；
- 采集器与设备未绑定、绑定停用、磁盘水位和会话配额拒绝；
- 并发请求不发生交叉写坏文件或完成状态越过。

### 5.4 完成门槛

模拟清单中的六类业务数据均可生成会话并以不同分片顺序写成与原文件 SHA-256 相同的临时文件；
本期不允许提前写入正式归档或 MySQL `archive_file`。

## 6. 第四期：校验归档、历史查询与清理

状态：已完成（2026-07-12）。实现、故障窗口和全量样本证据见 `第四期开发报告.md`。

### 6.1 生成内容

- `UPLOADING -> VERIFYING -> ARCHIVED/FAILED/EXPIRED` 状态转换；
- 位图完整检查、`stat` 大小检查、窗口化 `mmap` 和增量 SHA-256；
- 正式归档路径生成、父目录创建和同文件系统原子 `rename`；
- MySQL 归档元数据插入、唯一冲突读取和崩溃窗口恢复；
- Operator Token 保护的归档组合查询与详情；
- 基于 `(archived_at, archive_id)` 的稳定游标分页；
- 定时清理失败、过期和无会话旧临时文件，以及已结束 Redis 会话状态；
- 启动时只做必要的未完成状态扫描，不扫描或删除正式归档。

### 6.2 本期接口

- `POST /api/v1/uploads/{upload_id}/complete`，设备 HMAC；
- `GET /api/v1/archives`，Operator Bearer Token；
- `GET /api/v1/archives/{archive_id}`，Operator Bearer Token。

查询列表只返回元数据摘要，详情返回完整元数据但不读取文件正文。没有文件下载路由。

### 6.3 测试

- 缺片、大小不符、SHA-256 不符、映射窗口边界和合法大文件；
- 重复完成返回同一 `archive_id`；
- `rename` 后 MySQL 失败、MySQL 成功后 Redis 更新失败的恢复；
- 设备、工位、工单、产品 SN、类型、结果和 UTC 时间范围查询；
- 游标翻页无重复、无跳项，非法游标明确拒绝；
- 清理仅删除满足状态与时间条件的临时文件，正式归档始终不受影响。

### 6.4 完成门槛

`data/mock_inbox` 全部样本可归档，数据库大小/摘要与原始清单一致，查询能定位记录；按
`开发思路.md` 的崩溃矩阵执行故障注入后仍保持最多一个正式文件和一条归档记录。

## 7. 第五期：参考采集程序与三线仿真闭环

状态：代码、非网络测试和闭环 E2E 脚本已完成（2026-07-12）；本机网络提权额度拒绝启动
DataStream/collector 联合进程，最终闭环尚待一次权限复跑，详见 `第五期开发报告.md`。

### 7.1 生成内容

- C++11 参考采集程序，按配置管理三条线的设备目录和设备密钥；
- 临时名改名、稳定窗口和完成标志三种文件封口判定；
- 使用状态文件和原子 `rename` 的本地持久化 spool；
- 文件发现、稳定、上传、完成、成功和永久失败状态机；
- 对网络错误、超时和明确 5xx 的有上限退避，对永久 4xx 不重试；
- Python 模拟器扩展到三线 SPI/AOI/FCT 并发、追加、覆盖、延迟关闭和异常文件；
- 期望清单只用于测试核对，采集程序不得读取清单决定上传；
- IPC 断网、采集程序退出和重启恢复测试。

### 7.2 端到端链路

```text
Python 模拟器 -> 设备目录 -> C++ 采集程序 -> DataStream
                                              -> Redis/MySQL/正式归档
```

### 7.3 完成门槛

按 `模拟业务基线.md` 和 `采集端与文件生命周期契约.md`，三条线的检测结果、NG 图片、FCT
报告、设备导出报告和运行日志均可从产生过程走到正式归档；IPC 重启、8 小时断网模型、重复发现
和 Redis 会话丢失场景不会产生文件损坏或重复归档。

## 9. 阶段之间的变更规则

- 后一期可以修正前一期缺陷，但必须同步补回归测试和更新契约文档；
- 不为保持尚未发布的阶段内部接口而写兼容层；
- 数据库迁移一旦被下一阶段采用，不修改已执行迁移，而是新增下一版本迁移；
- 每期开始前复核实际输入边界、负责失败、调用方保证和拟加入的重试/兜底/抽象；
- 每期结束都复核 diff 和生成文件，阶段通过不等于生产发布；
- 用户可在任一期审查和调整外部契约，但不需要亲自完成剩余代码。
